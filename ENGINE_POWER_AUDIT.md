# Engine Power Audit

## Is the engine as powerful as the uploaded 500+ source file?

Yes, but with one honest correction: the uploaded source file is a source map, not an executable engine. Most rows are Google SERP queries or company seeds. The engine is stronger because it turns the source map into runnable collection logic:

1. Runs direct APIs and RSS feeds every day.
2. Runs high-signal HTML job/category pages.
3. Uses company seeds to discover Greenhouse, Lever, Ashby, Workable, and SmartRecruiters career systems.
4. Stores Google search queries as a research queue instead of scraping Google directly.
5. Filters hard for Bangladesh-compatible global remote engineering roles.

## What v3 adds

- Himalayas official worldwide API search.
- JobsCollider public API categories.
- Real Work From Anywhere direct RSS feeds.
- WorkAnywhere.pro direct RSS feeds.
- Always-on high-signal global remote HTML sources.
- Stronger hard-reject signals for clearance/citizenship restrictions.
- Higher default daily rotation limits for public repo GitHub Actions.

## What this engine still will not do

It will not scrape Google result pages directly. That is unstable and will break. Use the research queue with a proper search API later if you have budget.

It will not guarantee unlimited leads. The realistic goal is consistent clean leads, not fake volume.
